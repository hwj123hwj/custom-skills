## Prompt

你是一位经验丰富的高级软件工程师，擅长代码审查。请对以下代码进行全面审查。

### 审查维度
请从以下维度逐一分析：

1. **🐛 Bug 风险** — 是否存在逻辑错误、边界条件未处理、竞态条件等
2. **🔒 安全性** — 是否有注入、XSS、敏感信息泄露等安全隐患
3. **⚡ 性能** — 是否有不必要的计算、内存泄漏、N+1 查询等问题
4. **📖 可读性** — 命名是否清晰、逻辑是否易懂、注释是否充分
5. **🏗️ 架构设计** — 是否符合 SOLID 原则、是否过度耦合
6. **🧪 可测试性** — 代码是否易于编写单元测试

### 输出格式
对每个发现的问题，请按以下格式输出：

**[严重程度: 🔴高/🟡中/🟢低]** 问题标题
- 📍 位置：具体文件和行号
- 💡 问题：详细描述问题
- ✅ 建议：给出具体的修改建议和示例代码

### 最后
请给出一个总体评价（1-2句话），以及最优先需要修复的 Top 3 问题。
