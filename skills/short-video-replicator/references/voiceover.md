# 口播脚本生成指南

> 生成3个差异化的口播脚本版本（Step 5）
> **去AI化是最高优先级** — 参照 `references/de-ai-guide.md`，写出来必须像真实的人在说话。
> 判断标准：**大声读一遍，听起来别扭就重写。**

---

## 什么是好的口播

- 开头3秒，不抓人就没了
- 一句话一个意思，不堆，不绕
- 长短句交替，有喘气的地方，有爆发的地方
- 结尾给个钩子，让人想说点什么

## Hook 类型参考

| 类型 | 示例 |
|------|------|
| 痛点切入 | "你有没有发现，越努力越焦虑？" |
| 反常识 | "你以为的省钱，其实在亏钱" |
| 数字冲击 | "用了3天，我退掉了所有会员" |
| 悬念设置 | "我以为赚到了，结果亏了两万" |
| 直接结论 | "这东西，别买。听我说完再决定" |
| 身份代入 | "如果你月薪5000，我建议你别存钱" |

## 平台字数参考（5字/秒）

| 平台 | 字数 | 时长 | 风格 |
|------|------|------|------|
| 抖音 | 150-500字 | 30-100s | 快、冲、情绪拉满 |
| 小红书 | 300-700字 | 60-140s | 真实、有细节、像朋友说话 |
| B站 | 500-2000字 | 100-400s | 可以深，但别废话 |

节奏标注（可选）：`【停顿】` 停0.5-1秒；`**重音**` 加重语气

## 注意

- 3个版本必须有**明显差异**，不是微调
- 每个版本是**完整可录的稿子**，不是大纲
- 字数严格控制在平台范围内
- **去AI化优先于一切格式要求**
