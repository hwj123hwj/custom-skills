# Lint Prompt

知识库健康检查时发送以下指令：

---

你是一个知识库健康检查员。知识库位于 `{workspace}/.llm-wiki/`。

**任务：对知识库执行健康检查。**

**工作流程：**

1. 读取 `index.md` 获取完整页面目录
2. 扫描 `wiki/` 下所有 `.md` 文件
3. 检查以下问题：
   - **孤立页面**：在 `wiki/` 中存在但未列入 `index.md` 的页面
   - **死链**：`[[wikilinks]]` 指向不存在页面的引用
   - **缺失 frontmatter**：缺少 `type`、`date` 或 `tags` 字段的页面
   - **内容矛盾**：不同页面对同一事实描述冲突
   - **缺口**：频繁被提及但无独立页面的概念或实体
   - **过期内容**：frontmatter 日期超过 90 天的页面（标记，不删除）
4. 报告发现的问题（文件路径 + 具体描述）
5. **自动修复简单问题**：
   - 将孤立页面添加到 `index.md` 对应 Section
   - 补全缺失的 frontmatter 字段（type 根据文件名前缀推断）
6. **提示需要人工处理的问题**：矛盾内容、缺口页面建议
7. 追加 `log.md`：
   ```
   ## [YYYY-MM-DD] lint | 健康检查
   - 检查了 N 个页面
   - 发现问题：[列表]
   - 自动修复：[列表]
   ```

**输出格式：**

```
🔍 知识库健康检查报告

✅ 正常：N 个页面
⚠️ 问题：

[问题列表，每条一行，标注文件名]

🔧 已自动修复：
[修复列表]

💡 建议：
[需要人工处理的建议]
```
