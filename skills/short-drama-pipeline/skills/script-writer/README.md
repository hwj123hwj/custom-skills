# script-writer — 脚本撰写

基于确定的选题，生成一段**创意简报**——一段自然语言描述，直接喂给 libtv-skill 的 create_session.py。

## 核心原则

**不要在用户侧做创作。** libtv-skill 的后端 Agent 负责理解需求、拆解分镜、编排工作流、选模型、写 prompt。用户侧的职责是传话，不是创作。

具体来说：
- 不要自己写分镜表
- 不要自己编排镜头描述
- 不要添加"超写实风格，电影级光影，8K分辨率"之类的描述词
- 只需要清晰描述用户想要什么

## 输入

确定的选题 + 用户偏好（可选），例如：
- 选题："修仙逆袭"短剧
- 偏好：古风、3分钟以内、有反转

## 工作流

### Step 1: 整理创意简报

将选题转化为一段清晰的自然语言描述，格式如下：

```
{内容类型}，{风格}，{时长预期}。

{故事概要 / 内容描述，2-5 句话}

{补充要求（如有）}
```

**示例：**

```
一条AI短视频，古风修仙题材，30秒左右。

讲述一个被所有人看不起的废柴弟子，在悬崖边意外获得上古传承，觉醒后一招击败天才师兄的故事。节奏要快，开头就要有冲击力。
```

```
一个AI漫剧片段，日系动漫风格，1分钟左右。

校园恋爱场景，男主在樱花树下等女主，女主跑过来时摔倒被男主接住。要有少女漫的氛围感。
```

```
一段AI生成的产品展示视频，科技感风格，15秒。

展示一款无线耳机的使用场景：从开盒、佩戴到沉浸在音乐中的过程。画面要干净高级。
```

### Step 2: 确认

向用户展示创意简报，确认后进入 video-orchestrator。

## 模板

通用模板见 `templates/creative-brief-template.md`。

### 内容类型关键词

| 用户意图 | 发送给 libtv-skill 的关键词 |
|---------|--------------------------|
| 短剧/剧情 | "短剧"、"漫剧"、"AI短剧" |
| 单条短视频 | "短视频"、"AI视频" |
| 动画 | "动漫视频"、"AI动画" |
| 产品展示 | "产品视频"、"展示片" |
| MV | "音乐MV" |

### 风格关键词

让用户选择或从选题中推断，如：古风、现代、赛博朋克、日系动漫、写实、水墨、像素风等。

## 输出

创意简报保存到 `output/drafts/brief-{选题名}.md`。
